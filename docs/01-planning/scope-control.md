# Scope Control

This document keeps the project focused.

## Must Have

These items are required for the course project:

- Backend server with Express and TypeScript
- MongoDB database with Mongoose
- Flutter Android app
- Authentication with JWT
- Event CRUD from Android app
- Image upload to Firebase Storage
- Push notification with FCM

## Should Have

These items make the app match the EventHub design better:

- Search events
- Filter by category, date, location, and price
- Booking flow
- Bookmark events
- Notification history
- Organizer profile
- My tickets screen

## Could Have

These items are useful but can be added after the core requirements work:

- Map view
- Follow organizer
- Reviews
- Invite friend flow
- Share sheet
- Offline cache
- Dark mode
- Multi-language support
- CI/CD APK build

## Not In First Version

Avoid these until the required features are stable:

- Real online payment
- Complex chat system
- Advanced analytics dashboard
- Admin web panel
- Full social network features

## MVP Rule

The first complete version must prove the 4 main requirements before adding more UI polish.

Priority order:

1. Backend foundation
2. Authentication
3. Event CRUD
4. Image upload
5. Booking
6. FCM notification
7. Search/filter and UI polish
