# Testing And Deploy Guide

This guide prepares EventHub for a class demo or final submission.

## Local Verification

Backend:

```powershell
cd server
pnpm install
pnpm test
pnpm typecheck
pnpm build
```

Flutter:

```powershell
flutter analyze
flutter test
flutter build apk --debug
```

The debug APK is generated at:

```text
build/app/outputs/flutter-apk/app-debug.apk
```

## Render Deployment

The repository includes a root-level `render.yaml` Blueprint for a Node.js web
service.

Prerequisites:

- Git repository pushed to GitHub.
- MongoDB Atlas connection string.
- Cloudinary credentials.
- Firebase Admin service account values if real FCM push delivery is required.

Render steps:

1. Open Render Dashboard.
2. Choose **New** -> **Blueprint**.
3. Connect the GitHub repository.
4. Render detects `render.yaml` from the repository root.
5. Fill every `sync: false` environment variable.
6. Deploy the service.
7. Open `https://<service-name>.onrender.com/health` and confirm it returns `status: ok`.

Required Render environment variables:

```text
MONGODB_URI
FIREBASE_PROJECT_ID
FIREBASE_CLIENT_EMAIL
FIREBASE_PRIVATE_KEY
CLOUDINARY_CLOUD_NAME
CLOUDINARY_API_KEY
CLOUDINARY_API_SECRET
```

Generated or default variables from `render.yaml`:

```text
NODE_ENV=production
JWT_SECRET=<generated by Render>
JWT_EXPIRES_IN=7d
FCM_DRY_RUN=false
CLOUDINARY_FOLDER=eventhub/events
```

## MongoDB Atlas Notes

Use a MongoDB Atlas database for deployed demos.

Minimum setup:

1. Create an Atlas project and free/shared cluster.
2. Create a database user.
3. Allow network access for Render.
4. Copy the connection string.
5. Set it as `MONGODB_URI` in Render.

Example database name:

```text
eventhub
```

## Flutter With Deployed API

Run Android against the deployed backend:

```powershell
flutter run -d <device-id> --dart-define=API_BASE_URL=https://<service-name>.onrender.com/api
```

Build a debug APK against the deployed backend:

```powershell
flutter build apk --debug --dart-define=API_BASE_URL=https://<service-name>.onrender.com/api
```

For local Android emulator testing, keep using:

```powershell
flutter run -d <device-id> --dart-define=API_BASE_URL=http://10.0.2.2:4000/api
```

For Chrome/web local testing, the app defaults to:

```text
http://localhost:4000/api
```

## Demo Accounts

Create demo accounts through the app before the final demo:

```text
Organizer email:
Organizer password:

User email:
User password:
```

Use an organizer account to create/update/delete events. Use a normal user
account to book tickets, bookmark events, receive invitations, and test
notifications.

## Troubleshooting

- If Render health check fails, inspect Render logs and confirm `MONGODB_URI`.
- If image upload fails, confirm Cloudinary credentials.
- If push delivery fails but notification history is saved, confirm Firebase Admin credentials and Android `google-services.json`.
- If Android emulator cannot reach local backend, use `http://10.0.2.2:4000/api`.
- If Chrome cannot reach local backend, use `http://localhost:4000/api`.
